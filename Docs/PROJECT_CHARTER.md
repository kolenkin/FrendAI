# FrendAI Project Charter

## Mission
Create a personal AI engineering platform for researching, analyzing and developing ESP32-based and other embedded systems.

## Core principles

1. The project is more important than any single chat.
2. Knowledge must live in the repository.
3. Every engineering decision should be documented.
4. Automate repetitive work whenever possible.
5. GitHub is the single source of truth.

## Strategic goals

### Stage 1 — Infrastructure
- Git/GitHub
- VS Code
- Python
- ESP-IDF
- Documentation

### Stage 2 — Reverse engineering
- Flash dump analysis
- OTA analysis
- NVS analysis
- AI model extraction
- Automatic reports

### Stage 3 — Hardware knowledge
- Datasheets
- Pinouts
- Schematics
- Components

### Stage 4 — Own firmware and devices
Develop custom firmware and hardware based on accumulated knowledge.

### Stage 5 — AI Engineer
FrendAI should become an engineering assistant that understands the project, explains code and hardware, and helps develop new functionality.

## Long-term vision
The project should remain independent of any specific AI model. All knowledge should be stored in the repository so that any future AI assistant can continue development without losing context.
